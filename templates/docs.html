Failing Test Case Analyzer API

Welcome to the developer documentation for the Failing Test Case Analyzer.

Base URL

/ (served by FastAPI). Use the demo form or call the endpoints directly.

Endpoints

GET /

Returns the demo web page with form and frontend interface.

POST /analyze

Analyze provided question and code, returning a state object including generated failing test case and explanation.

Request JSON

{
  "question": "string",
  "user_code": "string",
  "baseline_code": "string",
  "failing_tc": "string (optional)"
}

Response JSON

The response echoes most request fields and adds analysis results. Key fields include:

  • failing_tc: generated or provided failing test case
  • verified_failing_tc, verified_simplified_tc, valid_tc: boolean flags (true/false)
  • final_explanation: raw explanation string from LLM
  • parsed_explanation: structured representation for frontend rendering

Example response:

{
  "question": "Print hello world ?",
  "user_code": "...",
  "baseline_code": "...",
  "failing_tc": "No specific input is needed, ...",
  "verified_failing_tc": true,
  "valid_tc": true,
  "final_explanation": "",
  "parsed_explanation": [ ... structured blocks ... ]
}

Unused fields (hidden in the demo) include language, message_stream_for_tc_refinement, and last_caller.
